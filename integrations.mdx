## Git Repository Syncing

Sync your documentation directly from a Git repository to keep content live and version-controlled.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to Settings > Integrations > Git Sync. Enter your repository URL, such as `https://github.com/your-org/shxian-qiu-docs`.
  </Step>
  <Step title="Authenticate" icon="key">
    Provide a personal access token with repo read permissions.
  </Step>
  <Step title="Configure Branch" icon="git-commit">
    Select the branch (e.g., `main`) and sync interval (e.g., every 5 minutes).
  </Step>
  <Step title="Enable Sync" icon="play">
    Toggle sync on. Changes push automatically on commit.
  </Step>
</Steps>

<Callout kind="tip">
  Use a dedicated docs branch to avoid disrupting your main codebase.
</Callout>

## API Access for Custom Integrations

Access Shxian Qiu programmatically using REST APIs at `https://api.example.com/v1`.

<ParamField path="docs" param-type="string" required="true">
  Documentation space identifier, e.g., `shxian-qiu`.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_TOKEN`.
</ParamField>

<CodeGroup tabs="JavaScript,Python">
  ````javascript
  const response = await fetch('https://api.example.com/v1/docs/shxian-qiu/pages', {
    headers: {
      'Authorization': 'Bearer YOUR_API_TOKEN',
      'Content-Type': 'application/json'
    }
  });
  const pages = await response.json();
  console.log(pages);
  ````

  ````python
  import requests

  headers = {
      'Authorization': 'Bearer YOUR_API_TOKEN',
      'Content-Type': 'application/json'
  }
  response = requests.get('https://api.example.com/v1/docs/shxian-qiu/pages', headers=headers)
  pages = response.json()
  print(pages)
  ````
</CodeGroup>

## Webhook Setup for Notifications

Set up webhooks to notify external services on events like page updates or new versions.

<Tabs>
  <Tab title="Slack" icon="message-circle">
    <Steps>
      <Step title="Create Webhook URL">
        In Slack, go to Incoming Webhooks and generate a URL like `https://hooks.slack.com/services/T00000000/B00000000/XXXXXXXXXXXXXXXXXXXXXXXX`.
      </Step>
      <Step title="Add to Shxian Qiu">
        Settings > Integrations > Webhooks > Add Slack. Paste the URL.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Discord" icon="discuss">
    Use a Discord webhook URL: `https://discord.com/api/webhooks/123456789/abcDEF`.
    
    Configure payload:
    
    ````json
    {
      "content": "New doc update in Shxian Qiu: {{page.title}}"
    }
    ````
  </Tab>
</Tabs>

## Export Options

Export your documentation for offline use or migration.

<Expandable title="Advanced Export Configurations" default-open="false">
  Customize exports with filters:
  
  | Format | Command | Options |
  |--------|---------|---------|
  | Markdown | `/export/md` | `--all-pages`, `--branch=main` |
  | PDF | `/export/pdf` | `--theme=dark` |
  | HTML | `/export/html` | `--include-images` |
  
  Run via CLI: `shxian export md --space=shxian-qiu --output=./docs`.
</Expandable>

<Callout kind="info">
  Exports respect your access permissions. API tokens grant granular control.
</Callout>
